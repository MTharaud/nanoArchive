# nanoArchive

This repository contains a set of Jupyter notebooks and data files for cleaning, analyzing and visualizing experimental results (fusion results, elemental analyses, clustering, etc.). The main content is organized as notebooks for data processing, visualization and exploration.

## Main files

- `1 - data_clean.ipynb`: Notebook for cleaning and preparing raw data.
- `2 - data_viz.ipynb`: Notebook for visualization (scatter, boxplots, barplots) and exporting figures into `Figures/`.
- `3 - data_clust.ipynb`: Notebook for clustering and group analysis.
- `4 - data_exam.ipynb`: Notebook for exploration and detailed examination of results.

Compressed data files (CSV, gzip):
- `fusion_results.csv.gzip`
- `fusion_results_merged.csv.gzip`
- `fusion_results_merged_with_clusters.csv.gzip`
- `fusion_results_cleaned.csv.gzip`

Other directories and files:
- `Add_data/`: Additional data and reference files (e.g. `Molar mass.csv`, `elements.txt`).
- `Data_spCal/`: Individual result files (files ending with `_results.csv`).
- `Figures/`: Output figures generated by the notebooks.
- `old/`, `Deleted/`: Archives or removed files kept for reference.
- `elements.txt`: List of elements used in analysis.

## Purpose

The repository centralizes and documents the processing of experimental results:

- Cleaning and harmonization of raw CSV files.
- Merging results into a single DataFrame.
- Visualizing elemental data by sample/depth and auxiliary parameters (conductivity, d18O, dD, etc.).
- Performing clustering of samples and interpreting the resulting groups.

## Prerequisites

Notebooks were written for Python 3.x and typically use the following libraries:

- pandas
- numpy
- seaborn
- matplotlib
- scikit-learn (likely used in clustering notebooks)

It is recommended to work inside a virtual environment (venv or conda).

Quick example to create a virtual environment and install common dependencies:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install pandas numpy seaborn matplotlib scikit-learn jupyter
```

If you prefer conda:

```bash
conda create -n nanoenv python=3.10
conda activate nanoenv
pip install pandas numpy seaborn matplotlib scikit-learn jupyter
```

## Usage

Open the notebooks using Jupyter Lab or Jupyter Notebook:

```bash
jupyter lab
# or
jupyter notebook
```

Then open one of the notebooks (for example `2 - data_viz.ipynb`). Notebooks expect the compressed `fusion_results_*.csv.gzip` files to be available at the repository root.

Notes specific to `2 - data_viz.ipynb`:
- The notebook loads `fusion_results_merged.csv.gzip` using:
  ```python
  merged_df = pd.read_csv('fusion_results_merged.csv.gzip', compression='gzip', sep=',', header=0, dtype={'# sample': str, 'Depth': str})
  ```
- Several plotting functions save figures into the `Figures/` directory. Make sure the directory exists (create it if missing):
  ```bash
  mkdir -p Figures
  ```

## Expected data structure

DataFrames typically contain columns such as:
- `Filename`, `Depth`, `Conductivity`, `# Non-soluble dust`, `d18O`, `dD`, `Core`, `# sample`, and many columns for elemental measurements (e.g. `Au`, `Fe`, `Sum`, ...).

Notebooks often convert `Depth` and `# sample` to numeric values and filter/sort by depth.

## Best practices

- Use a virtual environment to avoid dependency conflicts.
- Do not overwrite original compressed data files; work on copies or the cleaned file `fusion_results_cleaned.csv.gzip`.
- Prefer versioning notebooks or final scripts, and avoid committing large raw data files to the repository.

## Suggested additions

- Add a `requirements.txt` or `environment.yml` to pin dependencies.
